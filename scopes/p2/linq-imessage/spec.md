# P2 Scope 1: iMessage via Linq — Spec

> **Status:** FRD exists (pre-kickoff draft), spec to be generated

## Intent

Enable users to interact with Tryps through their existing iMessage group chats via Linq relay. Text-based trip status queries, activity suggestions, voting, RSVP, and expense logging — all without leaving iMessage.

## Key Questions (for Linq Architecture Kickoff)

- Webhook payload schema (exact JSON fields)
- Group chat identity mapping (stable conversation_id?)
- Programmatic group creation (auto-create iMessage group from trip?)
- Outbound message API (replies, proactive notifications, blasts)
- Burst rate limits (50 simultaneous messages for text blast?)
- Sandbox vs production differences
- Delivery status webhooks
