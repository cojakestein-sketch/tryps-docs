## What

Get real users testing Tryps and build the feedback loop that turns their experience into roadmap changes. TestFlight distribution, additional distribution channels, and a triage process for incoming feedback.

## Why

The product only gets better with real usage data. We need users in the app before April 2 — not after — so we can catch issues and validate that the experience works.

## Success Looks Like

- Active TestFlight users providing feedback
- Additional distribution channels beyond TestFlight established
- Feedback triage process: user report → GitHub issue → roadmap consideration
- Clear signal on what's working and what's broken before April 2
