### What We Built: Daily Trip Card for iMessage Groups

**Problem:** Send a visual trip summary in iMessage groups every morning so users can see trip status at a glance.

**Solution:** Three-part system:

#### 1. OG Image Generator (`supabase/functions/trip-card-image/`)

- Generates a 1200x630 PNG dynamically using Satori + resvg-wasm
- Shows: cover photo, Tryps logo, status badge ("11 days away" / "Happening Now"), trip name, dates, location, avatar stack, activity thumbnails, footer
- URL: `/functions/v1/trip-card-image?id={tripId}`

#### 2. Trip Card Web Page (`website-tryps/app/trip/[id]/page.tsx`)

- Next.js page deployed on Vercel (`website-tryps-sage.vercel.app`)
- Card 2 (detailed overview): trip name, flights, accommodation, vibes, expenses, packing progress, day-by-day itinerary timeline
- OG meta tags point to the image generator for link previews
- Env vars added to Vercel: `NEXT_PUBLIC_SUPABASE_URL`, `SUPABASE_SERVICE_ROLE_KEY`

#### 3. Daily Facilitator Update (`close-polls/index.ts` + `daily-facilitator.ts`)

- Updated to send 2 messages per trip per day:
  - **Message 1:** Trip card image (sent as Linq media attachment — shows inline, no "Click to Load Preview") + trip URL
  - **Message 2:** Text nudge based on completeness level
