4. Deep Dive: Agent Coordination — From Nudges to Workflow Engine

No topic response received from Jake's 3pm DM. Using the most pressing April 2 topic.

What It Is

The Tryps agent currently operates in "respond mode" — it answers questions, creates trips, and sends daily facilitation nudges. The April 2 gate requires "advance mode" — the agent proactively moves a trip from "just created" to "complete trip card with flights, stay, activities, and everyone confirmed." This is the difference between a chatbot and a concierge.

Why It Matters Now

This is the single biggest gap between current capabilities and the April 2 demo. Asif identified 5 technical challenges today (context window management, per-member state tracking, proactive sequencing, conflict resolution, external data integration). None of these are solved. The daily facilitator is text nudges — it needs to become a state machine that knows what's missing per-person and takes action.

Key Takeaways

  • Define "trip completeness" as a measurable score: Each trip card section (destination, dates, flights, stay, activities, attendees confirmed) gets a 0-1 completion score. The agent's job is to maximize the score, not just send reminders.
  • Build per-member tracking first: Current completeness is trip-wide. The agent can't say "Alex hasn't added flights yet" — it needs to. This is a database schema change (member_trip_progress table) + agent prompt change.
  • Sequencing matters more than intelligence: The agent doesn't need to be brilliant — it needs to ask for things in the right order. Dates first, then destination, then flights (once dates are locked), then stay, then activities. This is a hardcoded workflow, not AI creativity.
  • Conflict resolution = voting, not AI judgment: When 3 people want different dates, the agent shouldn't pick — it should create a poll, set a deadline, and enforce the result. The vote-on-behalf system Rizwan is building is the conflict resolution mechanism.
  • External APIs are Phase 2: For April 2, the agent can recommend destinations and activities from seed data. Real-time flight/hotel pricing via Duffel + booking via Stripe is Rizwan's travel-booking scope — it doesn't need to be in the coordination layer on day 1.

Resource

AI-Scientist-v2's agentic tree search — study how it branches through hypotheses and converges. Similar pattern to what the trip coordination engine needs.
