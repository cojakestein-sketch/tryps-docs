11. CS 101 — Jake's Daily Lesson

What is an API?

An API (Application Programming Interface) is a menu for software. When you walk into a restaurant, you don't go into the kitchen and cook — you look at the menu, order something specific, and the kitchen sends it back. An API works the same way: your app sends a request to another service ("give me flights from JFK to LAX on June 15"), and the service sends back a structured response with the data. The menu is the API — it tells you what you can ask for and how to ask.

Why it matters for Tryps: Almost everything Tryps does talks to an API. When Rizwan builds flight search, he's calling the Duffel API. When the iMessage agent adds an activity, it's calling the Supabase API. When Marty generates this report, he's calling the Brave Search API and the GitHub API. Every "integration" is really just two pieces of software reading each other's menus.

The aha moment: When Asif's iMessage-to-app sync is broken, it almost always means one of two things: the agent is ordering something that's not on the menu (wrong API call), or the kitchen got the order but didn't send it to the right table (the realtime subscription isn't listening). The "menu" metaphor is how your devs think about every bug.
