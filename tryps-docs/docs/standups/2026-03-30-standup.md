## 🛠 Tools for the Team

**Hidden Claude Code features you should know** — from Boris Cherny (creator of Claude Code at Anthropic). [Full thread →](https://x.com/bcherny/status/2038454336355999749)

Most useful for us:

1. **Mobile app** — Code from your phone. Claude app → Code tab. Good for quick changes without opening a laptop.
2. **`/teleport` + `/remote-control`** — Move sessions between mobile/web/terminal. Start on your phone, continue on desktop (or vice versa).
3. **`/loop` + `/schedule`** — Run Claude automatically on an interval (e.g., every 5min to babysit PRs, every 30min to address Slack feedback). Turn workflows into skills + loops.
4. **Hooks** — Deterministic logic in the agent lifecycle: auto-load context on session start, log bash commands, route permission prompts to WhatsApp.
5. **Chrome extension** — Essential for frontend work. Lets Claude verify its own UI output in a real browser and iterate until it looks right.
6. **`/branch`** — Fork your current session to try something without losing your place.
7. **`/btw`** — Side queries while the agent works. Ask quick questions without interrupting the current task.
8. **Git worktrees (`claude -w`)** — Run dozens of Claudes in parallel on the same repo. Essential for parallel work.
9. **`/batch`** — Fan out massive changesets across many worktree agents. Good for large migrations.
10. **`--add-dir`** — Give Claude access to additional folders/repos mid-session.
11. **`--agent`** — Custom agents with their own system prompt + tools. Define in `.claude/agents/`.
12. **`/voice`** — Voice input. Hold spacebar in CLI. Boris does most of his coding by talking.

**Bottom line:** If you're only using Claude Code as a chat-in-terminal, you're leaving a lot on the table. Try `/loop`, `/teleport`, and the Chrome extension this week.
