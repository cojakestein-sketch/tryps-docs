# Memory as a Service

## The Core Insight

We're selling memory as a service. Tryps holds the shared context between individuals — the markdowns related to the trip. That could not be more important.

## Why This Matters

We have a structural advantage for why users want to come to us: **we already have the shared context between the individuals.** No other platform holds the full picture of what a group decided, preferred, voted on, and experienced together.

This builds on the human-shared and selective preferences layer (Traveler DNA / Trip Vibe), but goes further — it's about **who holds the record of the trip itself.**

## The Data Moat

- Every trip generates structured context: preferences, votes, decisions, itineraries, splits, memories
- That context lives in our system, not scattered across iMessage threads and Google Docs
- The more trips a group takes together, the richer the shared memory becomes
- This is the flywheel: shared memory makes the next trip easier, which brings users back, which deepens the memory

## How It Connects to the Three Layers

1. **iMessage** captures the raw conversation and decisions
2. **Mobile app** structures and surfaces the shared context
3. **Agent layer** uses the accumulated memory to execute smarter on behalf of the group

The agent gets better because it has the memory. The memory gets richer because the agent facilitates more. That's the compounding loop.
