Bug Lifecycle

  • QA finds bug → ClickUp sub-ticket
  • Dev fixes → moves to "Ready for Testing"
  • QA verifies → "Ready for Production"
  • Still broken? → back to "To Do" with note

Test Case Files

QA writes a test case file per module before testing. Every scenario listed, marked pass/fail. The file is the source of truth for what's tested.
