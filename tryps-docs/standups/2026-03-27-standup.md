**1. Jake defined the April 2 gate: "text in an iMessage group to plan an entire trip end-to-end in natural language — stays, Spotify playlist, flights, everything — with the ability to book things related to the trip." You own the agent that does this. What can it do today vs. what's missing? Map the gap between current agent capabilities and that vision.**

What it can do today: Create trips from any message (name, destination, dates — all optional), add expenses/activities/flights/polls/voting, dress code management, daily facilitator with trip card previews, auto-registration for new users, memory extraction (user preferences from every message), trip card image generation + OG previews on jointryps.com, contact card sharing, retry logic for OpenAI failures.

What's missing for April 2: Agent proactively advancing trips (not just nudging — actually suggesting and booking), recommendations that auto-populate based on user memory, vote-on-behalf working end to end with batch DMs, cross-trip intelligence (learning from past trips to improve future ones), smooth handoff between iMessage agent and app (deep links working fully), agent memory influencing suggestions in real-time.

**2. Coordination/facilitation: the agent needs to advance a trip from "just created" to "complete trip card" — flights, stay, activities, everyone confirmed. Jake says this lives inside your existing scopes. What are the 5 biggest technical challenges? Jake is speccing this today and needs your input.**

1. Context window management — As trips get complex (20+ activities, 8 people, multiple polls), the system prompt with full trip snapshot exceeds token limits. Need smart summarization.
2. Multi-user state tracking — The agent needs to know who hasn't responded, who hasn't voted, who hasn't added flights — per person. Current completeness level is trip-wide, not per-member.
3. Proactive sequencing — Moving from level 1 to 10 requires knowing the right order to ask for things. Current daily facilitator is basic text nudges. Needs to become an actual workflow engine that adapts based on what's blocked.
4. Conflict resolution — When 3 people want different dates or different stays, the agent needs to facilitate consensus without being annoying. This is the hardest AI problem — knowing when to push vs back off.
5. External data integration — To auto-suggest flights, stays, and activities, the agent needs real-time pricing/availability APIs. Current ai_suggest is text-based. Needs structured data from Google Places, flight APIs, etc.
