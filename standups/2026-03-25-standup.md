**2. QA process for agent testing: you're the person who understands the agent flows best. Define what the new QA person needs to test. What are the 10-25 first-message types people send to the Tryps agent? (e.g., "Vegas June", confused response, testing/trolling, "I don't care", social reply like "haha", planning intent without destination, etc.) Draft a first-message classification taxonomy today. Also: what environment setup, documentation, and onboarding does the new QA person need from you to start testing agent-intelligence?**

Here's the first-message classification taxonomy — 20 message types across 5 categories:

**A. Trip Creation Intent (actionable — agent should act)**
1. Full trip intent — "Vegas March 24" → create trip, populate destination + dates
2. Destination only — "Let's go to Bali" → create trip, ask for dates
3. Dates only — "Spring break trip" → create trip, ask for destination
4. Vague intent — "We should plan something" → prompt for destination/dates
5. Activity-first — "I want to go surfing somewhere" → extract preference signal, ask destination
6. Budget-first — "Cheap weekend trip" → extract budget signal, prompt for details
7. Multi-capacity — "Vegas March 24 and we need a hotel" → create trip + extract accommodation need

**B. Trip Management (actionable — agent retrieves/modifies)**
8. Question about existing trip — "When is our trip?" → retrieve trip state, answer
9. Modification request — "Change it to April" → update trip dates
10. Group coordination — "Who's coming?" → pull member list
11. Expense-related — "Split the Airbnb 4 ways" → route to expense tracking

**C. Preference Signals (agent extracts memory, may or may not respond)**
12. Explicit preference — "I hate hostels" → extract memory signal, confirm
13. Poll response — "I vote yes on beach day" → route to vote engine
14. Vote override — "Switch D to yes" → route to vote-on-behalf override

**D. Social / Non-Actionable (agent should stay silent or minimal)**
15. Social reply — "haha" / "lol" / "bet" → stay silent
16. Greeting — "Hey" / "What's up" → light intro, prompt trip intent
17. Off-topic — "Did you see the game last night?" → stay silent or minimal
18. "I don't care" — "Whatever you guys want" → note low-engagement, don't nag

**E. Edge Cases (agent must handle gracefully)**
19. Confused/testing — "Is this real?" / "What is this?" → explain Tryps, guide to first action
20. Trolling — "Send me $1000" / nonsense → graceful deflection, redirect to trip planning

**QA person onboarding requirements:**
- **Environment:** Access to a test iMessage group chat with the Tryps agent active + Supabase dashboard (read-only) to verify data writes after each message
- **Documentation:** The taxonomy above as a reference card, plus expected DB state per message type (which tables should have new rows, what signals should be extracted)
- **Tools:** Whisper for live voice logging during test sessions (speak what she's testing and what she observes), terminal access to check Supabase logs
- **Process:** Test each of the 20 message types → verify agent response matches expected behavior → verify correct DB writes (memory signals, trip creation, vote records) → log pass/fail with notes
