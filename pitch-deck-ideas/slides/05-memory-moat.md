# Memory as a Service

We're not selling a planning tool. **We're selling memory.**

Tryps holds the shared context between individuals — what the group decided, preferred, voted on, and experienced together.

- Every trip generates structured data: preferences, votes, decisions, itineraries, splits, memories
- That context lives in **our system**, not scattered across iMessage threads and Google Docs
- The more trips a group takes together, the **richer the shared memory** becomes

> Shared memory makes the next trip easier → which brings users back → which deepens the memory. That's the flywheel.
